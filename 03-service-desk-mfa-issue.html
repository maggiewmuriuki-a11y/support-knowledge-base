User-Reported Issue:

"I cannot log into my work account this morning. After entering my password, it asks for an MFA code but my authenticator app is not generating any codes."

🔍 Initial Assessment

  • Issue started after user got a new phone
  • Old phone was factory reset — Authenticator not backed up
  • Password is correct but MFA step fails
  • User working remotely with urgent deadline

Impact: User locked out of Microsoft 365 (Outlook, Teams, finance system).

🛠️ Investigation Steps

Step 1: Verify User Identity

  • Verified via employee ID and security question
  • Confirmed with line manager
  • Checked Entra ID — account active

Step 2: Check MFA Registration

  • Authenticator linked to old device
  • No backup method configured

Step 3: Temporary Access

  • Issued Temporary Access Pass (TAP)
  • Validity: 1 hour, single use
  • Shared securely via Teams

Step 4: MFA Re-Registration

  1. Installed Microsoft Authenticator
  2. Visited https://aka.ms/mfasetup
  3. Added Authenticator App
  4. Scanned QR code
  5. Completed test authentication

Step 5: Verification

  • Login successful
  • Access to systems restored
  • TAP expired

✅ Resolution

Summary: MFA re-registered using TAP. Access restored within 3 hours 42 minutes.

Root Cause: Phone replaced without backup MFA or alternate method.
